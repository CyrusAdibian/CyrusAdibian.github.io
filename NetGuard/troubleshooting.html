راهنماها

راهنمای نصب عیب‌یابی محل نمایش خطاها خطاهای نصب خطاهای یکپارچگی داشبورد و رابط API نکات عمومی

عیب‌یابی

نت‌گارد در زمان بارگذاری افزونه، محیط وردپرس را بررسی می‌کند. اگر مشکلی وجود داشته باشد، پیام خطا همراه با راه‌حل مرحله‌به‌مرحله نمایش داده می‌شود.

محل نمایش خطاها

  • صفحات نت‌گارد — پنل کامل خطا یا هشدار
  • سایر صفحات مدیریت وردپرس — اعلان مدیریتی در بالای صفحه

خطاهای نصب و پکیج

نسخه PHP پایین است

علت

نسخه PHP سرور شما کمتر از 8.2 است.

راه‌حل
  1. در پنل هاست، نسخه PHP را به 8.2 یا بالاتر تغییر دهید.
  2. در محیط محلی (XAMPP/MAMP) نسخه PHP را به‌روزرسانی کنید.
  3. پنل وردپرس را رفرش کرده و نت‌گارد را دوباره باز کنید.

نسخه وردپرس پایین است

علت

نسخه وردپرس شما کمتر از 6.4 است.

راه‌حل
  1. از سایت نسخه پشتیبان تهیه کنید.
  2. از مسیر پیشخوان ← به‌روزرسانی‌ها وردپرس را آپدیت کنید.
  3. دوباره نت‌گارد را اجرا کنید.

پکیج افزونه ناقص است (vendor/ وجود ندارد)

علت

پوشه vendor/ در بسته بارگذاری‌شده وجود ندارد.

راه‌حل
  1. بسته کامل افزونه را دوباره دانلود کنید.
  2. پوشه فعلی افزونه را حذف و نسخه کامل را در wp-content/plugins/NetGuard/ کپی کنید.
  3. افزونه را غیرفعال/فعال کنید.

فایل‌های داشبورد موجود نیستند

علت

فایل‌های آماده داشبورد داخل پوشه افزونه پیدا نشده‌اند.

راه‌حل
  1. پکیج کامل انتشار را جایگزین کنید.
  2. از وجود فایل‌های dashboard/dist/index.js، dashboard/dist/style.css و dashboard/dist/index.asset.php مطمئن شوید.

جداول دیتابیس ساخته نشده‌اند

علت

نت‌گارد هنگام فعال‌سازی نتوانسته جداول موردنیاز را ایجاد کند.

راه‌حل
  1. افزونه را یک‌بار غیرفعال و دوباره فعال کنید.
  2. مجوزهای دیتابیس برای CREATE و ALTER را بررسی کنید.
  3. در صورت نیاز، WP_DEBUG را فعال و لاگ wp-content/debug.log را بررسی کنید.

نیاز به ionCube Loader

علت

این نسخه نت‌گارد به افزونه PHP با نام ionCube Loader نیاز دارد.

راه‌حل
  1. ionCube Loader را مطابق نسخه PHP نصب کنید.
  2. وب‌سرور/PHP را ری‌استارت کنید.

خطاهای یکپارچگی فایل

نت‌گارد فایل‌های افزونه را با integrity-manifest.json و خط مبنای قابل اعتماد مقایسه می‌کند.

مانیفست یکپارچگی پیدا نشد

علت

فایل integrity-manifest.json در پوشه افزونه وجود ندارد.

راه‌حل
  1. نسخه کامل افزونه را جایگزین کنید.
  2. افزونه را دوباره فعال کنید.

فایل‌های افزونه تغییر کرده‌اند

علت

برخی فایل‌ها با مانیفست انتشار یا خط مبنای معتبر همخوانی ندارند.

راه‌حل
  1. فایل‌های هسته افزونه را ویرایش نکنید.
  2. کل پوشه افزونه را با نسخه اصلی جایگزین کنید.
  3. افزونه را غیرفعال/فعال کنید تا خط مبنا دوباره ساخته شود.

هشدار مسیر ذخیره‌سازی اسکن قابل نوشتن نیست

علت

نت‌گارد امکان نوشتن در مسیر ذخیره‌سازی اسکن (معمولاً wp-content/uploads) را ندارد.

راه‌حل
  1. مجوز نوشتن مسیر uploads را اصلاح کنید.
  2. پس از اصلاح دسترسی، یک اسکن جدید اجرا کنید.

خطاهای داشبورد و رابط API

اگر صفحه باز می‌شود ولی پیام «مشکلی رخ داد» می‌بینید، معمولاً درخواست API ناموفق بوده است.

خطای نشست یا دسترسی (401/403)

راه‌حل
  1. صفحه را رفرش کنید.
  2. از حساب خارج و دوباره وارد شوید.
  3. سطح دسترسی مدیر را بررسی کنید.

مسیر REST پیدا نشد (404)

راه‌حل
  1. به تنظیمات پیوند یکتا بروید و یک‌بار ذخیره کنید.
  2. افزونه را دوباره فعال کرده و صفحه را رفرش کنید.

خطای سرور (500)

راه‌حل
  1. لاگ wp-content/debug.log را بررسی کنید.
  2. از نسخه صحیح PHP و وجود پوشه vendor/ مطمئن شوید.

پاسخ API مسدود یا نامعتبر

راه‌حل
  1. افزونه‌های امنیتی دیگر را موقتاً غیرفعال کنید.
  2. اجازه دسترسی مسیر /wp-json/ برای مدیر سایت را بررسی کنید.

خطای شبکه یا اتصال

راه‌حل
  1. اتصال اینترنت را بررسی کرده و دوباره تلاش کنید.
  2. باز بودن سایت وردپرس و دسترسی API را بررسی کنید.

نکات عمومی

  • همیشه از سایت نسخه پشتیبان داشته باشید.
  • پس از هر تغییر مهم، کش مرورگر و کش سایت را پاک کنید.
  • برای مراحل اولیه نصب، راهنمای نصب را ببینید.
